Welcome to your accessible template

This template demonstrates semantic structure, keyboard accessibility, clear focus indicators, and patterns that support WCAG 2.2.

Key accessibility features

  • Clear structure: Proper use of <header>, <nav>, <main>, <section>, and <footer> landmarks.
  • Keyboard support: All interactive elements are standard HTML controls or buttons, focusable and operable with the keyboard.
  • Visible focus: Strong focus indicators meet WCAG 2.2 focus requirements when styled appropriately.
  • Labels and instructions: Every form control has an associated label and optional hint text.
  • Reduced redundancy: Example patterns support minimizing re-entry of data (3.3.7 Redundant Entry) when implemented on multi-step flows.
